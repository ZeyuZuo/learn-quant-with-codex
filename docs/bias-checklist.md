# 回测偏差检查清单

本清单用于课程 9.1 和 Capstone 报告复查。它只服务学习和研究记录，不构成投资建议。

## 使用方式

每次展示策略回测结果前，至少回答下面 8 个问题，并把答案写进报告限制章节。

| 检查项 | 需要确认的问题 | 如果忽略会怎样 |
| --- | --- | --- |
| Look-ahead bias | signal 是否使用了当时还不知道的数据？ | 回测可能偷看未来，曲线被美化。 |
| Signal / position lag | 当天收盘产生的 signal 是否至少滞后一日进入 position？ | 策略可能吃到现实中无法成交的当天收益。 |
| Survivorship bias | 样本是否只包含今天仍然存在的股票？ | 失败或退市资产被排除，历史表现偏乐观。 |
| Selection bias | ticker、行业、时间段是否因为结果好看才被选中？ | 研究问题被结果反向决定。 |
| Data snooping | 是否反复尝试大量规则，只展示最好结果？ | 偶然赢家被误认为有效规律。 |
| Transaction costs | 是否写清 commission、slippage 和 turnover？ | 高频换仓策略会被不现实地抬高收益。 |
| Rebalancing assumptions | 多资产权重和再平衡频率是否明确？ | 组合收益无法复查，成本也无法估计。 |
| Reporting boundary | 结论是否避免“未来会盈利”这类承诺？ | 学习报告会被误读成投资建议。 |

## Capstone 写法模板

```text
本报告使用历史样例数据进行学习型回测。我们检查了 signal 到 position 的滞后、交易成本、参数扫描、样本内 / 样本外切分和常见回测偏差。即使样本外结果较好，它仍然只是历史验证，不代表未来收益，也不构成投资建议。
```

## 最低验收

- 至少列出 4 类偏差。
- 至少说明 1 个本项目已经用测试或代码规则防住的偏差。
- 至少说明 1 个本项目仍然没有覆盖的现实问题，例如流动性、税务、真实订单延迟或心理压力。
